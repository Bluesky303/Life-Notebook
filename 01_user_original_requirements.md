# 用户原始需求（逐字记录）

你现在在我仓库的根目录，需要你新建一个项目的文件夹，我准备把后端和网页端放在一起（先做这个），其他端之后开新仓库。前端可以选择nextjs, 我希望后端是python fastapi，数据库可以按你说的来。ai层面考虑多接口，为了可用你需要先接入codex，也就是自己开一个codex新会话用来作为一个上下文这种。部署的话制作docker可以先不做。作为用户，我不考虑后端的组织之类的，我希望前端能有科幻简洁的感觉。首先考虑主页面，左上角添加总财产的显示，可以点击进入资产管理。然后主页的顶栏可以进入其他页面，中间先放上时间表，时间表展示一周的时间安排，之后提到的任务会自动按时间填入任务栏并且展现空闲时间，需要按时间段块状地填入，需要使用不同颜色区分重要性。在下方则是“动态”部分，指的是日常生活动态，动态部分的顶部是一个输入框，可以快捷进行多种记录（后面会提到不同页面的记录）和快捷分类，最好可以语音输入。下方则是最近的动态。接下来考虑其他页面，1. 财产记录：需要考虑多种钱包以及资产，在主页面仅显示标记为现金的总额，目前有微信钱包，支付宝钱包，和支付宝“理财”部分，这里是指进行理财投资。需要记录投入资产，以及收益，每日收益目前依靠手动输入，应该在周一到周五使用任务表提醒查看和记录。需要绘制折线图表示趋势。然后钱包部分需要记录每日的支出和收入，以及具体是什么东西的收入和支出（也就是分类，比如吃饭，买书等）。需要能按照分类查看收入支出，需要能够看到每个月的收入支出饼图。2. 顶栏进入任务栏，任务栏记录任务信息和规划等，有多种分类，需要考虑多种时间排布，比如每天每周每月每年，包括庆祝生日使用的农历每年，甚至是零散的某几天的时间段，还有具体某段时间的每天每周什么的等。需要记录任务的信息，甚至通过ai来进行初步的规划。需要方便修改，需要能够分类查看，需要重要性（和分类不同），重要性使用颜色区分3. 顶栏进入知识库，他同时作为博客和词条百科的类型进行，所以会类似于博客和wiki的形式，词条会记录新鲜事物，比如新的芯片这种。博客会记录解决问题的过程。以上至少支持markdown，当然简单的latex也可能需要，然后方便修改。4. 顶栏进入选项，可能的一些设置，比如模型选择，页面主题什么的。接下来你先把以上话原封不动记录在一个文档，然后建立新文档总结以上内容并且写成计划书，需要规划任务步骤，规划测试和运行，然后在这里告诉我你的修改意见。
